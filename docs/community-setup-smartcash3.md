# SmartCash 3.0.0 Community Setup

This guide shows copy/paste-friendly setup commands for public SmartCash 3.0.0
nodes, SeedNodes, CPU miners and wallets. It does not contain private keys,
RPC passwords or server credentials.

## Network

- Protocol: `90300`
- P2P port: `29678`
- RPC port: `29679` local only
- SAPI port: `28080` for SmartNodes
- Explorer: `http://151.252.59.32/explorer/`
- Bootstrap: `http://151.252.59.32/bootstrap/smartcash3-bootstrap-4269526.tar.gz`
- Fixed public peers: `151.252.59.32:29678`, `151.252.59.33:29678`

Recommended public 5-CT topology:

- Public seed + explorer node
- SeedNode
- CPU miner node
- CPU miner node
- Additional full node seed without mining

## Requirements

Seed/full node:

- Ubuntu Server `24.04 LTS` x86_64
- 2 vCPU or better
- 4 GB RAM minimum, 8 GB recommended
- 80 GB free SSD storage minimum, 120 GB recommended
- Public IPv4 address
- Open inbound TCP `29678`

SmartNode:

- Ubuntu Server `24.04 LTS` x86_64
- 2 vCPU or better
- 4 GB RAM minimum, 8 GB recommended
- 80 GB free SSD storage minimum, 120 GB recommended
- Public static IPv4 address
- Open inbound TCP `29678` and `28080`
- Exactly `100000 SMART` collateral in the controller wallet
- One SmartNode private key generated by the controller wallet

Build host for desktop/core wallets:

- Ubuntu `24.04 LTS` x86_64
- 4 vCPU recommended
- 8 GB RAM minimum, 16 GB recommended
- 40 GB free storage for Linux builds, 80 GB for Windows cross-builds

## 1. Build SmartCash Core 3.0.0 On Ubuntu 24.04

Run as a normal sudo-capable user:

```bash
sudo apt update
sudo apt install -y git build-essential libtool autotools-dev automake pkg-config bsdmainutils curl ca-certificates python3 make g++ libevent-dev libboost-system-dev libboost-filesystem-dev libboost-chrono-dev libboost-test-dev libboost-thread-dev libminiupnpc-dev libzmq3-dev
git clone https://github.com/SmartCashCMTY/MiningNode.git SmartCash3
cd SmartCash3
TAR_OPTIONS=--no-same-owner make -C depends -j2
./autogen.sh
./configure --prefix="$(pwd)/depends/x86_64-pc-linux-gnu" --disable-tests --disable-bench
make -j2 src/smartcashd src/smartcash-cli src/smartcash-tx
sudo install -m 0755 src/smartcashd /usr/local/bin/smartcashd
sudo install -m 0755 src/smartcash-cli /usr/local/bin/smartcash-cli
sudo install -m 0755 src/smartcash-tx /usr/local/bin/smartcash-tx
```

Verify:

```bash
smartcashd --version
smartcash-cli --version
```

## 2. Install A Seed/Full Node

This is a public full node. It does not enable SmartNode mode and does not enable
block production.

```bash
sudo useradd --system --home /var/lib/smartcash3 --shell /usr/sbin/nologin smartcash || true
sudo install -d -m 0750 -o smartcash -g smartcash /var/lib/smartcash3
sudo install -d -m 0755 /etc/smartcash3
RPCPASSWORD="$(openssl rand -hex 32)"
EXTERNAL_IP="$(curl -fsS4 https://ifconfig.me)"
sudo tee /etc/smartcash3/smartcash.conf >/dev/null <<EOF
daemon=1
server=1
listen=1
txindex=1
maxconnections=128
port=29678
rpcport=29679
rpcbind=127.0.0.1
rpcallowip=127.0.0.1
rpcuser=smartcashrpc
rpcpassword=${RPCPASSWORD}
externalip=${EXTERNAL_IP}:29678
addnode=151.252.59.32:29678
addnode=151.252.59.33:29678
EOF
sudo chown root:smartcash /etc/smartcash3/smartcash.conf
sudo chmod 0640 /etc/smartcash3/smartcash.conf
sudo tee /etc/systemd/system/smartcash3.service >/dev/null <<'EOF'
[Unit]
Description=SmartCash 3.0 daemon
After=network-online.target
Wants=network-online.target

[Service]
User=smartcash
Group=smartcash
Type=forking
PIDFile=/var/lib/smartcash3/smartcashd.pid
ExecStart=/usr/local/bin/smartcashd -conf=/etc/smartcash3/smartcash.conf -datadir=/var/lib/smartcash3 -pid=/var/lib/smartcash3/smartcashd.pid
ExecStop=/usr/local/bin/smartcash-cli -conf=/etc/smartcash3/smartcash.conf -datadir=/var/lib/smartcash3 stop
Restart=on-failure
RestartSec=10
TimeoutStartSec=600
TimeoutStopSec=120

[Install]
WantedBy=multi-user.target
EOF
sudo ufw allow 29678/tcp
sudo systemctl daemon-reload
sudo systemctl enable --now smartcash3
```

Check sync:

```bash
smartcash-cli -conf=/etc/smartcash3/smartcash.conf -datadir=/var/lib/smartcash3 getblockcount
smartcash-cli -conf=/etc/smartcash3/smartcash.conf -datadir=/var/lib/smartcash3 getconnectioncount
```

Optional bootstrap before first start or after stopping the service:

```bash
sudo systemctl stop smartcash3
cd /tmp
curl -LO http://151.252.59.32/bootstrap/smartcash3-bootstrap-4269526.tar.gz
curl -LO http://151.252.59.32/bootstrap/smartcash3-bootstrap-4269526.tar.gz.sha256
sha256sum -c smartcash3-bootstrap-4269526.tar.gz.sha256
sudo tar -C /var/lib/smartcash3 -xzf smartcash3-bootstrap-4269526.tar.gz
sudo chown -R smartcash:smartcash /var/lib/smartcash3
sudo systemctl start smartcash3
```

## 3. Install A SmartNode

First prepare the VPS exactly like the seed/full node above, then change the
config to SmartNode mode.

On the controller wallet, generate one SmartNode key:

```bash
smartcash-cli smartnode genkey
```

Send exactly `100000 SMART` to a new address in the controller wallet. Wait for
the required confirmations, then get the collateral output:

```bash
smartcash-cli smartnode outputs
```

On the SmartNode VPS, replace `REPLACE_WITH_SMARTNODE_PRIVKEY` with the key from
`smartnode genkey`:

```bash
sudo tee -a /etc/smartcash3/smartcash.conf >/dev/null <<'EOF'
sapi=1
sapiport=28080
smartnode=1
smartnodeprivkey=REPLACE_WITH_SMARTNODE_PRIVKEY
EOF
sudo ufw allow 28080/tcp
sudo systemctl restart smartcash3
```

Check the VPS daemon:

```bash
smartcash-cli -conf=/etc/smartcash3/smartcash.conf -datadir=/var/lib/smartcash3 getblockcount
smartcash-cli -conf=/etc/smartcash3/smartcash.conf -datadir=/var/lib/smartcash3 snsync status
smartcash-cli -conf=/etc/smartcash3/smartcash.conf -datadir=/var/lib/smartcash3 smartnode status
```

On the controller wallet, add one line to `smartnode.conf`:

```text
SN01 YOUR_VPS_IP:29678 SMARTNODE_PRIVKEY COLLATERAL_TXID COLLATERAL_OUTPUT_INDEX
```

Example format only:

```text
SN01 203.0.113.10:29678 93ExamplePrivateKey txidexample 0
```

Restart the controller wallet, wait until it is synced, then start the alias:

```bash
smartcash-cli snsync status
smartcash-cli smartnode start-alias SN01
smartcash-cli smartnode list full
```

Expected final state: `ENABLED`.

## 4. Core Wallet Builds

Linux Qt wallet:

```bash
cd SmartCash3
make -j2 src/qt/smartcash-qt
strip src/qt/smartcash-qt
./src/qt/smartcash-qt
```

Windows cross-build from Ubuntu:

```bash
sudo apt update
sudo apt install -y g++-mingw-w64-x86-64 mingw-w64-x86-64-dev nsis zip
cd SmartCash3
make clean || true
TAR_OPTIONS=--no-same-owner make -C depends HOST=x86_64-w64-mingw32 -j2
./configure --prefix="$(pwd)/depends/x86_64-w64-mingw32" --host=x86_64-w64-mingw32 --disable-tests --disable-bench CC=x86_64-w64-mingw32-gcc-posix CXX=x86_64-w64-mingw32-g++-posix CXXFLAGS="-std=c++11"
make CC=x86_64-w64-mingw32-gcc-posix CXX=x86_64-w64-mingw32-g++-posix -j2
```

Download Core wallet binaries only from GitHub releases:

```text
https://github.com/SmartCashCMTY/Node-Client-Wallet/releases/tag/v3.0.0
```

For the Windows Core wallet, select `smartcash3-3.0.0-x86_64-w64-mingw32.zip` from the
GitHub release and verify it against the `SHA256SUMS` file in the same release.
Do not download wallet binaries from `151.252.59.32`; that host is only used
for explorer/bootstrap services.

## 5. BeeHive Desktop Light Wallet

BeeHive is a SAPI light wallet. It does not run a local daemon and does not sync
the blockchain.

Requirements:

- Node.js `18.x`
- npm `9.x`
- 8 GB RAM recommended for builds

Build:

```bash
git clone https://github.com/SmartCashCMTY/beehive-desktop.git
cd beehive-desktop
npm install --legacy-peer-deps
npm run build:linux
npm run build:win
```

Artifacts are created in `dist/`:

- Linux AppImage: `BeeHive SmartCash 3-3.0.0.AppImage`
- Windows x64 ZIP: `BeeHive SmartCash 3-3.0.0.zip`

Published BeeHive Desktop Light Wallet binaries are downloaded only from GitHub:

```text
https://github.com/SmartCashCMTY/beehive-desktop/releases/tag/v3.0.0
```

## 6. Install A CPU Mining Node

Use this role for community nodes that should mine blocks without SmartNode
collateral:

```bash
sudo bash contrib/smartcash3/install-node.sh miner
```

If prompted, enter the payout address. Otherwise write it to `/etc/smartcash3/miner.env`:

```bash
PAYOUT_ADDRESS=YOUR_PAYOUT_ADDRESS
MIN_BLOCK_HEIGHT=4269520
MIN_CONNECTIONS=1
MINING_CPU_QUOTA=10%
```

The miner timer uses the normal `generatetoaddress` RPC path, does not require a
local wallet, does not require an official mining-signature key and pays rewards
to `PAYOUT_ADDRESS`. The recovery update keeps mining-signature support available
but does not enforce signed blocks by default. During each mining RPC call, the
timer applies `MINING_CPU_QUOTA` as a runtime limit to `smartcash3.service` and
removes it again when the call exits, so normal full-node operation and reindexing
are not permanently throttled.

## Security Notes

- Never publish `wallet.dat`, private keys, `smartnodeprivkey`, RPC passwords or
  `.env` files.
- Keep RPC bound to `127.0.0.1` unless you know exactly what you are doing.
- Use a firewall and only open public ports `29678` and, for SmartNodes, `28080`.
- The SmartNode collateral remains in the controller wallet. Do not copy the
  controller wallet to the VPS.
